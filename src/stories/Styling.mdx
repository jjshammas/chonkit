## Block sizing

Units in Chonkit are based on "blocks", which are square units that scale with the `blockSize` prop on `ChonkitProvider`. This allows you to create pixel-perfect layouts that align to a pixel grid.

See below for an example of how block size impacts components (using the same font in this example):

<Canvas of={StylingStories.AllBlocks} />

When designing your interface, consider whether a unit quantity is best expressed in blocks or pixels. For example, you may want to use pixel values for all positioning properties to ensure consistent spacing regardless of block size:

```tsx
{
	"border": 1, // will always be 1 block
	"padding": "8px" // will always look like 8px, regardless of block size
}
```

…will create the following effect. Notice how the border scales with block size, but the padding remains visually consistent:

## Sx prop

Chonkit components support the `sx` prop for applying custom styles using a CSS-in-JS approach. The `sx` prop accepts an object where keys are CSS properties (in camelCase) and values are the corresponding CSS values.

```tsx
<Box sx={{ backgroundColor: "primary", padding: 10 }}>Custom styled box</Box>
```

The `sx` prop is a **superset** of the standard `style` prop.

### Grid snapping

Numbers expressed in pixels within the `sx` prop will automatically snap to the nearest block grid defined by the `blockSize` prop on `ChonkitProvider`. This ensures that all dimensions align perfectly to the pixel grid.

```tsx
<ChonkitProvider blockSize={5}>
	<Box sx={{ padding: "12px" }}>Padding will be snapped to 10px</Box>
</ChonkitProvider>
```

### Colors

Any color values provided in the `sx` prop will automatically resolve to the current theme's color palette.

```tsx
sx: {
	backgroundColor: "primary",
	color: "primary.fg",
}

// Theme definition
palette: {
	primary: {
		main: "#3b5bdb",
		fg: "#ffffff",
	},
}
```

### Interactions

Interaction states (like hover and active) are supported through the `sx` prop using pseudo-selectors:

```tsx
sx: {
	backgroundColor: "primary",
	"_hover": {
		backgroundColor: "primary.dark",
	},
	"_active": {
		backgroundColor: "primary.darker",
	},
	"_focus": {},
	"_disabled": {},
}
```

### Pixelified properties

Certain CSS properties that affect pixel rendering are automatically "pixelified" using the `sx` prop. This ensures that effects like dithering and pixelated borders are applied correctly. This includes properties like:

- `borderRadius`
- `boxShadow`
- `background` (for gradients)

For more examples, view the Box component documentation.

### Chonkit-specific properties

Components may define additional custom properties within the `sx` prop that are specific to Chonkit. For example, the `Box` component supports a `depth` property to apply a 3D effect. These are declared as `sx` properties instead of top-level props to work alongside theming and state effects.

### Responsive design

Chonkit supports responsive design through breakpoint objects in the `sx` prop. Any style property can accept an object with breakpoint keys (`xs`, `sm`, `md`, `lg`, `xl`, `2xl`) to define values at different viewport widths.

```tsx
<Box
	sx={{
		backgroundColor: {
			xs: "#ff6b6b",
			sm: "#4ecdc4",
			md: "#45b7d1",
			lg: "#96ceb4",
			xl: "#ffeaa7",
		},
		padding: { xs: "10px", sm: "15px", md: "20px", lg: "30px" },
		borderRadius: { xs: 4, sm: 8, md: 12, lg: 16 },
	}}
>
	Responsive content
</Box>
```

Breakpoints are defined in the theme and can be customized:

```tsx
// Default breakpoints (in pixels)
breakpoints: {
	xs: 0,    // Base/mobile
	sm: 640,  // Small tablets
	md: 768,  // Tablets
	lg: 1024, // Laptops
	xl: 1280, // Desktops
	"2xl": 1536, // Large screens
}
```

Responsive values work for **all** properties in the `sx` prop, including:

- Standard CSS properties (colors, spacing, typography, etc.)
- Chonkit-specific properties (`depth`, `borderRadius`, `backgroundGradient`, etc.)
- Interaction states can also use responsive values

## Fonts

Chonkit does not ship with any default fonts. Use traditional CSS properties in the `sx` prop to apply fonts to your components, or set a font globally:

```csss
.chonkit-root {
	--ck-text-font-family: "Habit", monospace;
	--ck-text-font-size: 16px;
	--ck-text-line-height: 16px;
	--ck-text-adjustment-top: 0;
	--ck-text-adjustment-left: 0;

	--ck-text-font-family-sm: "Habit Compact", monospace;
	--ck-text-font-size: 16px;
	--ck-text-line-height: 16px;
	--ck-text-adjustment-top: 0;
	--ck-text-adjustment-left: 0;
}
```

Fonts that are designed for pixel use cases typically expect to be rendered at a 16px font size, hence this is the default font size for both the regular and "small" font variables. However, you can adjust these values as needed for your design.

The `-text-adjustment` properties can be used to fine-tune the vertical and horizontal alignment of text to ensure pixel-perfect rendering. This is often necessary when using custom fonts that may not align perfectly to the pixel grid by default.

This Storybook documentation uses "Tiny5" as a default font, which is a free pixel font available on Google Fonts.

If you are using a grid size greater than 1.5, you may find that most pixel fonts are designed for much smaller block grids and can look too large. In this case, you may want to choose to use a font that does not perfectly map to the grid.
